# UI ロードマップ

「多くのユーザーに使われる、とても使いやすいリッチな UI」を目指すための投資計画。
優先領域は ①ダッシュボードの可視化 ②一覧・テーブル UX ③フォーム体験（2026-06 決定）。
デザイントークン・色・状態アイコンは `docs/design-system.md` を正とする。

## Phase 1: アカウント発行 UI（前提機能）

登録 → ログインの輪を閉じる。仕様は sereni-work-api の `docs/account-provisioning.md` を正とする。

- 職員・利用者の詳細画面に「アカウント」カード（状態バッジ / ログイン ID / 発行・再送・再発行）
- 利用者の発行結果ダイアログ: ログイン ID + 初期パスワードの一度きり表示、コピー操作、再表示不可の注意書き
- 一覧にアカウント状態バッジを追加

## Phase 2: 一覧・テーブル UX

毎日触る画面の体験を最優先で磨く。

- TanStack Table v8 + shadcn/ui Data Table パターンへ移行（利用者 / 企業 / 割当 / 日報 / 最適化の各一覧）
- 列ソート・テキスト検索・ステータスの faceted filter（複数選択チップ）
- フィルタ・ページ状態は URL クエリと同期（nuqs）— リロード・共有しても状態が残る
- ローディングはスケルトン行、0 件は空状態コンポーネント（説明 + 主要アクションへの CTA）
- 行クリックで詳細へ。行内に主要アクション（日報: レビューへ等）

## Phase 3: ダッシュボード可視化

ホーム画面を「今日やることが分かる」画面にする。

- **要対応キュー**（最優先・画面上部）: 未レビュー日報 / NEEDS_ACTION / 面談希望（interview_needed）/ 本日のプレチェック未提出。各項目から対象へ直接遷移
- 統計カード: 進行中実習数 / 本日の日報提出率 / 確定待ち割当 など
- チャート（shadcn/ui charts = Recharts）:
  - 日報提出率の推移（直近 2〜4 週）
  - プレチェック体調トレンド（condition / fatigue / anxiety の平均推移）
  - 実習ステータス分布
- 配色はチャートもセマンティックトークン（success / warning / danger）に従う

## Phase 4: フォーム体験

- 利用者登録・編集: セクション分割（基本情報 / 希望・スキル / 配慮事項 / 通勤条件）+ ページ内ナビ
- 配慮事項・スキル・希望職種はタグ入力 UI（既存データからのサジェスト付き）
- バリデーションは `mode: "onTouched"` でリアルタイム化、送信中はボタンをローディング状態に
- 成功・失敗トースト（sonner）、未保存変更がある場合の離脱ガード

## 横断（各 Phase に含めて進める）

- 画面遷移・状態変化のマイクロアニメーション（tw-animate-css の範囲で控えめに）
- キーボード操作・ARIA は `docs/accessibility.md` 準拠を維持
- iOS のリッチ UI（今日の実習ビュー・日報入力体験）は別途 sereni-work-ios 側で計画する
