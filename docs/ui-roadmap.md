# UI ロードマップ

「多くのユーザーに使われる、とても使いやすいリッチな UI」を目指すための投資計画。
優先領域は ①ダッシュボードの可視化 ②一覧・テーブル UX ③フォーム体験（2026-06 決定）。
デザイントークン・色・状態アイコンは `docs/design-system.md` を正とする。

## Phase 1: アカウント発行 UI（前提機能）✅ 完了

登録 → ログインの輪を閉じる。仕様は sereni-work-api の `docs/account-provisioning.md` を正とする。

- 利用者詳細に「アカウント」カード（状態バッジ / ログイン ID / 発行・再発行）
- 利用者の発行結果ダイアログ: ログイン ID + 初期パスワードの一度きり表示、コピー操作、再表示不可の注意書き
- 職員管理画面（`/staff`）: 登録ダイアログ + 招待メール送信・再送 + アカウント状態バッジ

## Phase 2: 一覧・テーブル UX ✅ 完了

毎日触る画面の体験を最優先で磨く。

- TanStack Table v8 + 共通 `DataTable`（`components/shared/data-table.tsx`）へ移行（利用者 / 企業 / 割当 / 日報 / 自動提案 / 職員）
- ステータス等の絞り込みは `FacetedFilter`（単一選択チップ。サーバー側フィルタのため複数選択は API 対応とセットで検討）
- フィルタ・ページ状態は URL クエリと同期（既存 `useListParams` を使用。同等機能のため nuqs は導入しない）
- ローディングはテーブル形のスケルトン行、0 件は `EmptyState`（説明 + 主要アクションへの CTA）
- 行クリックで詳細へ。職員一覧は行内アクション（招待メール送信）
- 列ソート: API の `sort` / `order` パラメータ（各モジュールでキーをホワイトリスト定義）+ `SortHeader`（昇順 → 降順 → 解除、URL 同期）。対象列: 利用者・職員=氏名 / 企業=企業名・受け入れ人数 / 割当=期間 / 日報=日付

## Phase 3: ダッシュボード可視化 ✅ 完了

ホーム画面を「今日やることが分かる」画面にする。API は `GET /dashboard`（sereni-work-api の dashboard モジュール）で一括集計する。

- **「今日の対応」キュー**（画面上部）: 要対応日報 / 面談希望 / 確認待ち日報 / 本日のプレチェック未提出（利用者ごとに割当詳細へ遷移）。0 件の項目は表示せず、全て 0 なら完了表示
- 統計カード: 実習中 / 実習予定（確定済み）/ 本日の日報提出（提出数 / 想定数）
- チャート（shadcn/ui chart = Recharts、`--chart-1〜5` トークンで配色）:
  - 日報の提出状況（直近 2 週の提出想定数と提出数の棒グラフ）
  - プレチェック体調トレンド（condition / fatigue / anxiety の平均推移、1〜5）
  - 実習ステータス分布（ドーナツ）
- 提出想定数は「期間がその日を含む IN_PROGRESS / COMPLETED の割当数」で近似（割当の状態履歴は持っていないため）

## Phase 4: フォーム体験 ✅ 完了

- 利用者フォーム: セクション分割（基本情報 / 希望・スキル / 配慮・通勤 / その他）。企業フォームも同様（基本情報 / 受け入れ条件 / 連絡先・実習情報）
- 配慮事項・スキル・希望職種・必要スキルはタグ入力 UI（`components/shared/tag-field.tsx`。Enter・読点で確定、× / Backspace で削除）
- バリデーションは `mode: "onTouched"` でリアルタイム化、送信中はボタンをローディング表示に
- 成功・失敗トースト（sonner）、未保存変更の離脱ガード（`use-unsaved-changes-guard.ts`。App Router の制約によりブラウザ操作のみ対象）
- タグのサジェスト: `GET /tag-suggestions`（利用者・企業の登録済みタグを頻度順に集計）を TagField の datalist 候補として表示（希望職種・スキル・配慮事項）

## 横断（各 Phase に含めて進める）

- 画面遷移・状態変化のマイクロアニメーション（tw-animate-css の範囲で控えめに）
- キーボード操作・ARIA は `docs/accessibility.md` 準拠を維持
- iOS のリッチ UI（今日の実習ビュー・日報入力体験）は別途 sereni-work-ios 側で計画する
