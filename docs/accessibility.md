# アクセシビリティ方針（Web）

アクセシビリティは追加機能ではなく基本設計に含める。WCAG を意識した設計とする。

## 必須対応

- キーボード操作のみで全操作が完了すること
- スクリーンリーダー対応
- フォーカス管理（モーダル表示時のトラップ、ページ遷移時のフォーカス移動）
- 適切な aria 属性（ランドマーク、ライブリージョン、状態の通知）
- 高コントラスト
- フォームエラーの読み上げ（エラーと入力欄の関連付け）
- 色覚に配慮した表示。色だけに依存しない表現（アイコン・ラベル併用）

## 実装の指針

- shadcn/ui（Radix UI ベース）のアクセシブルなプリミティブを優先して使う
- セマンティック HTML を基本とし、div + onClick のような実装をしない
- テーブル・カレンダー等の複雑な UI もキーボードナビゲーションを確認する
- axe 等による自動チェックを Component / E2E テストに組み込む
